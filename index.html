      • 重置几何体纹理坐标

      • BufferGeometry设置纹理坐标uv

        该类是一个 几何模型(Geometry) 的高效替代，因为它使用缓存（buffer）来保存所有数据，包括顶点位置、面索引、法向量、颜色、UVs以及自定义属性。 这节约了向GPU传递全部这些数据的成本。但同时也使得BufferGeometry要比 几何模型(Geometry) 更难以处理，不是以对象的方式来访问，比如使用Vector3来访问位置数据， 以Color对象来访问颜色数据，你得从相应的attribute缓存中访问原始数据。 这使得BufferGeometry很适合用来存储静态对象，也就是当我们创建完模型实例后不太需要去操作它。
        WebGL 把和每个顶点模型关联的数据保存在 属性（attributes） 中。 比如顶点位置、顶点法向量、顶点颜色等。当使用 几何模型(Geometry) 时，renderer 负责封装这些信息到有类型数组缓存中并发送该数据到着色器。 当使用BufferGeometry时，所有这些和独立属性关联的数据都被保存进缓存中。 举例来说，这意味着为了获取一个顶点的关联位置数据，你必须调用 .getAttribute 来访问 'position' attribute，然后可以访问该位置属性的x/y/z坐标值。

      • 加载包含uv的外部模型

        ObjectLoader：用于加载JSON资源的加载器。和 JSON加载器(JSONLoader)不同，这个加载器使用对象的 .type 属性来映射到它们的原有类。

    • 数据纹理对象DataTexture
      • DataTexture (直接从原始数据和宽高来创建纹理。)

        DataTexture( data, width, height, format, type, mapping, wrapS, wrapT, magFilter, minFilter, anisotropy )
        data参数必须是一个ArrayBuffer或者类型数组。 其它参数继承自基类 纹理(Texture), magFilter 和 minFilter 都默认设置为 THREE.NearestFilter。flipY 和 generateMipmaps 属性默认设置为false。
        数据的解析取决于类型（type参数）和格式（format参数）: 如果type为THREE.UnsignedByteType, 那么就使用Uint8Array来获取纹理单元。 如果format为THREE.RGBAFormat, 那么data里面每4个值代表一个纹理单元（texel）; Red, Green, Blue 和 Alpha (透明度)。类似的，THREE.RGBFormat 指定用3个值来表示一个纹理单元。
        对于包装的类型，THREE.UnsignedShort4444Type, THREE.UnsignedShort5551Type 或者 THREE.UnsignedShort565Type，一个纹理单元的所有颜色分量可以通过在一个Uint16Array整数元素里的位域来获取。
        要使用类型 THREE.FloatType 和 THREE.HalfFloatType, WebGL 的实现必须分别支持扩展 OES_texture_float 和 OES_texture_half_float。要对这些类型的双线性插值使用THREE.LinearFilter，必须有WebGL扩展 OES_texture_float_linear 和 OES_texture_half_float_linear 的支持。

      • 创建RGBA数据

        在设置材质时必须将MeshPhongMaterial的transparent属性设置为ture，允许透明，否则贴图的RGBA中的A的值会无效。

  • 相机对象(投影方式) 正投影相机和透视投影相机解释
    • 正投影相机(OrthographicCamera)：角度不同投影长度不同

      OrthographicCamera( left, right, top, bottom, near, far )
      left — 相机视椎体（Camera frustum）左面。
      right — 相机视椎体（Camera frustum）右面。
      top — 相机视椎体（Camera frustum）上面。
      bottom — 相机视椎体（Camera frustum）下面。
      near — 相机视椎体（Camera frustum）前面（靠近相机的这一面）。
      far — 相机视椎体（Camera frustum）后面（远离相机的这一面）。

      .setViewOffset ( fullWidth, fullHeight, x, y, width, height )
      fullWidth — 多视图设置的全宽
      fullHeight — 多视图设置的全高
      x — 副摄像头的水平偏移
      y — 副摄像头的垂直偏移
      width — 副摄像头宽度
      height — 副摄像头高度
      该方法用于在一个较大的视椎体中设置视图偏移。这对于多窗口或多监视器/多机设置是有用的。关于如何使用它，你可以查看 远景相机(PerspectiveCamera) 中的示例。

      clearViewOffset ()
      清除视图偏移。

      updateProjectionMatrix ()
      更新相机投影矩阵，必须在参数发生变化后调用。

    • 透视投影相机(远景相机)PerspectiveCamera：位置不同投影长度不同

      进行远景投影的相机。 远景投影，也称之为透视投影。这个是我们人眼观察世界的模式，

      PerspectiveCamera( fov, aspect, near, far )
      fov — 相机视锥体垂直视角
      aspect — 相机视锥体宽高比
      near — 相机视锥体近裁剪面
      far — 相机视锥体远裁剪面。

      属性（Properties）
      #.fov：相机视锥体垂直视角，从下到上的观察角度。
      #.zoom：获取或设置相机缩放因子（zoom factor）。
      #.near：相机视锥体近裁剪面。
      #.far：相机视锥体远裁剪面。
      #.focus：焦点，用于立体视觉和场深度效果的目标距离。这个参数不影响投影矩阵，除非是使用立体摄影机。这个参数不影响投影矩阵，除非是使用立体摄影机。
      #.aspect：相机视锥体宽高比，等于窗口宽度除以窗口高度。
      #.view：视椎体窗口规格或null。
      #.filmGauge：用于较大坐标轴的胶片尺寸。默认为35（毫米）。这个参数不影响投影矩阵，除非.filmOffset设置为非零值。
      #.filmOffset：水平偏离中心偏移量，和.filmGauge同单位。
